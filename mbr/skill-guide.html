FNPL Monthly Business Review Skill

An AI agent skill for Cursor that auto-generates a complete FNPL Monthly Business Review slide deck from Databricks data — including interactive charts, data tables, and a full-screen HTML presenter — in minutes instead of hours.

What Does the Skill Do?

The MBR skill is a Cursor AI agent skill that connects to Databricks, executes read-only SQL queries against the FNPL base table, and produces a self-contained HTML slide deck. The agent handles everything end-to-end: data extraction, validation, chart rendering (Plotly.js), table formatting, and final assembly into a single-file slide presenter.

Just say “run mbr” and the skill takes care of the rest.

How Does the Skill Work?
1

Data Extraction

Connects to Databricks via MCP, runs read-only SQL queries against the FNPL base table. All queries are parallelized for speed.

2

Validation

Cross-checks row counts, date ranges, and metric values against prior months to catch data anomalies before building slides.

3

Template-Driven Build

Reads 5 self-contained HTML templates (slides 9–13) and replaces {{PLACEHOLDER}} tokens with computed data. CSS is ID-scoped per slide.

4

Static Slide Assembly

Builds Cover, Agenda (timeline layout), Executive Summary, and GA 2.0 slides from reference HTML in the skill definition.

5

Combined HTML Output

Assembles a full-screen slide presenter (1280×720) with keyboard, touch, and swipe navigation. Single HTML file, no server needed.

6

Auto-Archive

Pushes the deck to GitHub Pages dashboard and the Intuit enterprise git repo. Each month’s deck is preserved and never overwritten.

Key Interactive Features
✎

Edit / Present Mode

Toggle between presentation and live-editing mode in-place. No page reload — all DOM changes persist across mode switches.

+

Add Slide

Insert new slides from 4 MBR-styled templates (Title, Key Takeaway, Section Header, Blank). New slides match the exact design of data slides.

🗑

Delete Slide

Remove any slide with a confirmation dialog. The presenter auto-updates dot indicators, counter, and navigation.

💾

Save for Everyone

Commits the current deck to GitHub via API. Uses serializeCleanHTML() to strip edit-mode artifacts before saving.

▶

Full-Screen Presenter

Arrow keys, swipe, dot navigation. Responsive scaling on smaller screens. Dark navy background with smooth slide transitions.

📝

Inline Editing

Elements marked data-editable become contenteditable in Edit Mode. Team members can update text directly in the browser.

Limitations
  • Requires Databricks access and authentication via the Databricks MCP server configured in Cursor.
  • Currently generates FNPL product slides only. PCA, RAD, and TTFA require separate skills or manual additions.
  • Charts use the Plotly.js CDN — an internet connection is needed on first load (cached thereafter).
  • Save for Everyone requires a GitHub Personal Access Token (PAT) with repo write permissions.
  • Executive Summary card text is manually sourced from Slack or team input — it is not auto-generated from data.
  • Template and Query changes require updating both the build script (fnpl_mbr_apr26_build.py) and SKILL.md to stay in sync.
  • The base table name is required from the user running the skill — the agent asks for it during Step 0 setup and does not assume a default.
How to Import This Skill
1

Copy the skill folder

Clone or copy the skill directory into your Cursor skills folder:

# From the shared repo cp -r skills/fnpl-mbr-slide9/ ~/.cursor/skills/fnpl-mbr-slide9/
2

Verify folder contents

The skill folder must contain all of these files:

~/.cursor/skills/fnpl-mbr-slide9/ SKILL.md ← skill definition (agent reads this) template.html ← Slide 9 (Application Profile) template-slide10.html ← Slide 10 (Funded Loans) template-slide11.html ← Slide 11 (Monthly Performance) template-slide12.html ← Slide 12 (DPD30+ by Vantage) template-slide13.html ← Slide 13 (DPD30+ by Term) queries.md ← Slide 9 SQL queries queries-slide10.md ← Slide 10 SQL queries queries-slide11.md ← Slide 11 SQL queries queries-slide12.md ← Slide 12 SQL queries queries-slide13.md ← Slide 13 SQL queries slide11-table-spec.md ← table layout specification slide13-chart-spec.md ← chart layout specification schema-reference.md ← column mappings & table quirks intuit-ecosystem-white.svg ← Intuit logo for cover slide verify_fnpl_mbr_html.py ← post-build HTML verifier
3

Configure Databricks MCP

Ensure the Databricks MCP server is enabled in your Cursor settings. The skill uses the execute_sql tool for all data queries and authenticate for initial connection.

4

Run the skill

Open Cursor’s agent chat and type:

run mbr

The agent will ask a few setup questions (MBR month, cutoff date, product order), then execute the full pipeline automatically. Output lands in ~/Downloads/FNPL_MBR_<MonYY>.html.
